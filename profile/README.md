# 프로젝트 개요
### 서비스 이름
링크룸(Linkroom) - 청년의 이야기와 다시 연결되는 방
![LinkRoom 홈 화면](./링크룸 홈.png)
### 한 줄 소개
말해도 되는 공간, 연결되는 다음 단계.
링크룸은 청년의 목소리를 이해하고, 정책 기회•사람과 다시 이어주는 AI 상담 파트너.

### 해결하려는 문제
	1.	상담 기록 누락
	•	상담사는 대화를 듣고 기록까지 병행해야 하며 일부 정보는 문서화 과정에서 빠짐
	2.	상담 이후 행정·정리 부담
	•	요약, 분류, 보고 작성까지 모두 수작업
	3.	정책 추천·위기 탐지와 같은 후속 활용은 사실상 어려움
	•	상담 데이터가 있어도 “의미로 정리된 구조화 데이터”가 아님
	4.	청년의 말이 ‘상담실을 벗어나면 사라지는 문제’
	•	상담사가 바뀌면 맥락이 끊기고, 상담 후 행동으로 이어지지 못함

### LinkRoom이 제안하는 변화
	•	AI가 상담 음성을 실시간으로 이해
	•	감정·키워드·상황을 의미 단위로 추출
	•	상담 일지를 자동 생성
	•	청년의 상황 기반으로 정책·기회 RAG 추천
	•	상담 결과를 하나의 Re:Connect Report로 제공하여
  - ‘대화 → 회복 → 행동’로 이어지는 상담 경험을 만든다.

## 기술 스택 계획

### **Frontend**
- React / Flutter 
- UI 구성: 상담 화면, STT 결과, 요약결과, 문서 미리보기, 대시보드

### **Backend**
- FastAPI (Python) 또는 Node.js(Express)
- REST API 기반 데이터 전달 구조

### **AI Model**
- Whisper (Small)  
- Gemma 등 요약 LLM  
  
### **Database**
- MongoDB / PostgreSQL / Supabase 중 택1

### **Infra**
- GitHub Repository  
- HuggingFace / Colab / Local GPU (모델 개발용)
